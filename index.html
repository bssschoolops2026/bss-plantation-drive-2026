Settings

Connect live team sync · configure alerts & weekly digest

Live team sync (Firebase)
This app already points at its shared project — your teammates don't need to configure anything. They just open the app and click "Sign in with Google" with their own @bh.edu.pk or @beaconhouse.net account.
Access is restricted to @bh.edu.pk and @beaconhouse.net accounts, enforced two ways: the app rejects and signs out any other Google account immediately, and — this is the part that actually matters for security — the Firestore rules below check the signed-in email's domain server-side, so someone couldn't bypass the app and hit your database directly with an unauthorized account.
Settings is further restricted to one admin account (mahnoor.safdar@bh.edu.pk). Everyone else can't see the Settings tab at all, and — enforced the same server-side way — the rules below also block anyone but that account from editing clusters or the alert/digest preferences, even if they tried to hit the database directly.
Manage clusters

Add, edit, or remove clusters — no more editing source code. Changes sync live to your team when Firebase is connected, or stay saved on this device otherwise.

Manage route costs

Set an estimated cost per truckload for each route in the Dispatch Plan, so the team can see a rough logistics cost per load — and a running total — before committing to a route.

Trash

Deleted a dispatch or cluster by mistake? Nothing is gone for good — restore it here.

Weekly email digest (GitHub Actions)
A scheduled job in this project's GitHub repo emails the digest every week — it runs on GitHub's servers, independent of anyone opening this dashboard.

One-time setup:
1. In Google Account → Security → 2-Step Verification → App passwords → create one for "Mail"
2. In the GitHub repo → Settings → Secrets and variables → Actions → New repository secret:
  GMAIL_USER = your Gmail address, GMAIL_APP_PASSWORD = the 16-character app password
3. Same page → Variables tab → New repository variable:
  DIGEST_RECIPIENTS = comma-separated emails, FIREBASE_PROJECT_ID = your Firebase project ID
The workflow file (.github/workflows/weekly-digest.yml) is already in the repo — nothing else to do.
